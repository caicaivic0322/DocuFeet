# 架构草案

## 目标

做“基层医生副手”，不做自动诊断系统。

## 当前链路

1. 医生上传检查单照片，并填写症状/补充病情。
2. 后端先跑红旗规则，识别高危词。
3. 后端把图片、规则命中、病情文本打包给 Ollama。
4. 模型输出结构化 JSON。
5. 若规则命中高风险，则后端可以覆盖模型风险等级，确保安全优先。

## 推荐后续演进

- OCR 层：PaddleOCR / 苹果原生 Vision / 轻量商用 OCR
- 检验标准化：LOINC 别名映射
- 知识库：指南、公开药品标签、转诊标准
- 路由策略：主模型负责通用解释，MedGemma 负责复杂医学推理
- 审计：保存输入、检索证据、模型版本、输出结果
