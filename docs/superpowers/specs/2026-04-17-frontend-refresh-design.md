# 前端轻改版设计说明

## 背景

当前项目已经具备完整的首页与工作台骨架，能够展示产品定位、模型状态、病例录入与结构化输出结果。但前端整体气质仍偏向技术原型：

- 首页叙事更像 README 的可视化呈现，而不是面向基层医生或医院管理者的正式产品页面。
- 文案强调技术实现与阶段说明较多，弱化了基层场景价值、使用信任与产品使命感。
- 视觉上已经有卡片、渐变和玻璃质感，但品牌识别不够集中，部分样式仍带有模板残留痕迹。

本次改版不调整核心页面结构，不改动产品能力边界，也不修改前后端接口。目标是在现有信息架构上完成一次“轻改版”升级，让页面更像一款可对外展示的医疗 AI 产品首页。

## 目标

本次改版聚焦以下四个结果：

1. 让首页第一屏更有叙事张力，能够让基层医生或医院管理者快速理解产品站位。
2. 让整体视觉更有发布型产品质感，同时保留医疗场景应有的克制和可信。
3. 让模块文案从工程视角转为场景视角，突出“帮基层医生更快识别风险和下一步动作”。
4. 让工作台看起来更像正式产品界面，而不是功能演示面板。

## 非目标

- 不新增前端功能模块。
- 不重构页面结构和路由。
- 不调整表单字段、请求参数、接口协议和结果数据结构。
- 不加入新的动效系统或复杂交互。
- 不处理 OCR、患者版输出、知识库接入等路线图能力。

## 受众与语气

### 主要受众

- 基层医生
- 乡镇医院与县级医院管理者

### 语气要求

- 有叙事感，但不夸张。
- 有产品主张，但不过度营销。
- 强调基层医疗场景理解，而不是强调模型先进性。
- 所有核心表述都应回到“辅助判断、提示风险、明确下一步”，避免“替代医生”倾向。

## 设计方向

本次采用“发布型质感 + 更强叙事”的轻改方案。

页面整体保留当前的单页结构：

- 顶部导航
- 首屏 hero
- 运行状态条
- 能力卡片
- 工作台
- 安全原则
- 路线图

但每个区域的表达逻辑会重新梳理，使页面阅读顺序更符合对外展示：

1. 先建立产品立场与场景共鸣。
2. 再证明产品如何进入真实基层流程。
3. 再展示工作台能力。
4. 最后明确安全边界与后续规划。

## 内容策略

### 1. 首屏

首屏要从“技术能力介绍”转为“产品立场表达”。

调整原则：

- 保留“不是 AI 医生，而是乡镇医生的副手”这一核心定位，但升级为更自然、更具画面感的表述。
- 标题、副标题、按钮文案都应减少工程味，增强产品发布页质感。
- 强调产品解决的是基层医生面对检验单、症状和风险判断时的信息整理与优先级判断问题。
- 模型状态模块继续存在，但在视觉上作为可信度证明，而非首屏主角。

目标效果：

- 用户进入页面后先感受到“这是一款理解基层场景的产品”，再注意到技术实现方式。

### 2. 状态条与能力卡片

现有模块会从“首版输入 / 首版输出 / 推理方式”这类内部视角，转成更接近用户价值的表达。

调整原则：

- 卡片标题要从“我们现在做了什么”改成“它能在哪个关键节点帮上忙”。
- 描述文案强调真实场景：检查单拍照、风险分级、下一步动作、转诊优先级。
- 保留本地部署和模型可用状态这些信息，但表述方式更产品化。

目标效果：

- 让医院管理者看到落地可行性。
- 让医生看到使用入口和价值闭环。

### 3. 工作台

工作台保持现有双栏结构与字段配置，但通过文案和层级优化，让其更像正式产品。

调整原则：

- 输入侧文案强调“最少必要信息录入”，降低复杂感。
- 上传区、按钮、空状态、辅助说明都要更简洁、更可信。
- 输出侧强调“医生版结论”是帮助判断的结构化参考，而不是最终诊断。
- 风险等级、医生总结和下一步行动需要更清楚的视觉层级。

目标效果：

- 让用户可以把它直接理解为一个即将投入试点的产品界面。

### 4. 安全原则与路线图

这两个模块继续保留，但文案要从“项目说明”改成“产品承诺”与“演进路径”。

调整原则：

- 安全原则文案更像输出协议与产品边界声明。
- 路线图文案减少纯技术路线表达，更多强调能力如何一步步进入真实临床流程。

目标效果：

- 让用户感受到产品既有边界感，也有明确成长方向。

## 视觉策略

### 保留项

- 现有明亮底色与卡片式版面
- 轻量渐变背景
- 圆角、半透明、柔和阴影
- 响应式单页结构

### 优化项

- 将整体配色从偏通用科技蓝，调整为更有医疗产品辨识度的暖冷平衡方案。
- 强化标题排版张力，降低普通宣传页感。
- 收口卡片层级、描边、阴影和背景噪声，让页面更完整。
- 减少“模板组件”观感，提升品牌一致性。

### 视觉原则

- 首屏更有存在感，但不能花哨。
- 整体要安静、清晰、可信。
- 动效维持少量点缀，不新增复杂运动。
- 在移动端保持层级清楚，不因为轻改而引入拥挤感。

## 实施范围

本次允许修改的主要文件：

- `frontend/src/App.tsx`
- `frontend/src/index.css`

本次不计划依赖的文件：

- `frontend/src/App.css`

如无必须，将把旧模板残留样式继续闲置，不在本轮中作为视觉系统的一部分使用。

## 测试与验收

### 验收标准

- 页面结构和功能保持可用。
- 文案整体更加统一，明显减少技术说明书感。
- 首屏、卡片、工作台、原则与路线图在视觉上属于同一套设计语言。
- 页面在桌面和移动端都能正常阅读，不出现明显溢出或错位。
- 构建和静态检查通过。

### 验证方式

- 本地执行前端构建。
- 本地执行 lint。
- 人工检查关键页面区块在不同宽度下的层级和文案表现。

## 风险与取舍

- 因为本次不重构结构，叙事能力提升会主要依赖文案与视觉收口，而不是信息架构重组。
- 因为不增加新功能，页面的“可信度证明”仍以状态检查、结构化输出和安全边界为主。
- 为保证医疗产品分寸，视觉上不会追求过强品牌张扬感，而是以“像正式发布产品”优先。

## 实施建议

实现时建议按以下顺序推进：

1. 重写首页文案与模块标题。
2. 调整全局视觉变量与背景氛围。
3. 收口首屏、卡片、工作台和结果区的层级样式。
4. 清理无关样式噪声并完成响应式回归检查。
